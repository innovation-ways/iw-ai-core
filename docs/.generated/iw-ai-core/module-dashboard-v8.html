FastAPI Dashboard

The dashboard is a FastAPI web UI (port 9900) providing real-time visibility and manual controls for the IW AI Core orchestration platform. It is the primary human interface for managing projects, work items, batches, documentation, code understanding, and background jobs.

Purpose

The dashboard surfaces everything the daemon writes to PostgreSQL in a browsable, interactive UI. Agents run in git worktrees and report results via iw CLI commands; the dashboard reads that same state and renders it. No business logic lives in the dashboard routers — they only validate inputs and delegate to the orch/ layer.

Key Components

Application Factory

dashboard/app.py — create_app() bootstraps the FastAPI application:

  • Lifespan: On startup, mark_orphaned_runs() flips crashed TestRuns to error, then verify_instance_identity() confirms the DB fingerprint matches (CR-00014). Failures are suppressed in test/operator contexts but propagate in production.
  • Session cookie: Sets an iw_chat_session HttpOnly cookie (readable by JS for localStorage scoping) if absent.
  • Alembic guard middleware: Throttles DB-schema-behind warnings at most once per 10 seconds.
  • Schema-mismatch handler: Catches SQLAlchemy ProgrammingError / UndefinedColumn and returns a clear 503 with remediation steps (F-00079).
  • Jinja2 templates: Registered on app.state.templates with custom filters: intcomma, timeago, fmt_ts_time, localdt. The is_db_stale global is also attached.
  • Health endpoint: GET /health — used by browser_verification steps.

Routers

All routers are registered in app.py:277–308. Business logic stays in orch/; routers are thin.

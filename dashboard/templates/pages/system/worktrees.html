Worktree Health

Git status of all active agent worktrees across all projects. Dirty or orphaned worktrees can block squash-merges — resolve before the merge phase.

{% include "fragments/worktree_table.html" %}

Status guide

clean — no changes, safe to merge
dirty — tracked files modified; merge will fail
untracked — new files not staged; harmless but worth reviewing
orphaned — worktree on disk with no active batch item; safe to delete
no_path — path in DB does not exist on disk
error / timeout — git command failed

To remove an orphaned worktree: delete via the item detail page or git worktree remove --force <path>, then click Prune orphans to run git worktree prune across all projects so the dashboard no longer shows stale entries.

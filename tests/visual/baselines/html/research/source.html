R-00071 — Opencode Dashboard Embedding

Type: Research Version: v1.0 Status: Draft Created: 2026-05-15

Summary

This research document evaluates the feasibility of embedding the OpenCode agent directly within the IW AI Core dashboard, enabling in-browser agent interaction without leaving the dashboard URL. It covers protocol requirements, security considerations, and recommended implementation paths.

Motivation

Currently, agents run in standalone terminal sessions. Developers must context-switch between the dashboard (for queue/status) and a terminal window (for agent interaction). Embedding the agent in the dashboard would collapse this into a single interface, reducing cognitive overhead and making agent progress visible in the same window as the work queue.

Technical Approach

The recommended approach is a WebSocket bridge between the dashboard backend and the opencode subprocess:

# Pseudocode — WebSocket → opencode bridge
class OpenCodeBridge:
    def __init__(self, worktree_path: Path):
        self.process = subprocess.Popen(
            ["opencode", "--chat"],
            cwd=worktree_path,
            stdin=subprocess.PIPE,
            stdout=subprocess.PIPE,
        )
    
    async def handle_ws(self, websocket):
        async for msg in websocket:
            self.process.stdin.write(msg + "\n")
            await self._drain_stdout(websocket)

IW AI Core — Architecture Overview

Type: Architecture Version: v1.4 Status: Published Generated: 2026-05-24

Executive Summary

IW AI Core is an AI orchestration platform that drives LLM-agent development across multiple git worktrees. It schedules agents in background jobs, aggregates their output, and surfaces a web UI for code understanding, documentation, research, tests, and batch control.

System Components

The platform consists of four primary components:

  • Orchestration Database (PostgreSQL) — stores all work items, batch state, daemon events, and job results. The single source of truth.
  • Daemon — polls the database every 60 s, picks approved batches, creates git worktrees, launches LLM agents (opencode or claude-code), runs fix cycles, and squash-merges to main.
  • Dashboard (FastAPI) — human interface on port 9900. Provides queue, history, batches, code browsing (RAG), docs, research, tests, jobs, and worktree views.
  • iw CLI — the agent-to-DB bridge. Agents call iw step-done to record results. All operational state lives in PostgreSQL — no files, no race conditions.

Data Flow

  1. Human approves a batch of work items via the Dashboard or CLI.
  2. Daemon picks up the batch, acquires a row-level lock (FOR UPDATE), and creates a git worktree at a fixed path.
  3. The agent runs inside the worktree, calls iw step-done, and exits.
  4. Daemon detects completion, squash-merges the worktree branch to main, and marks the batch COMPLETE.

Known Limitations

  • Worktree names are derived from the CR number — collisions if two batches of the same CR run concurrently (mitigated by daemon locking).
  • Per-worktree DB exists only when ai-dev/iw-config/ is present in the project.

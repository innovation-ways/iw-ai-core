# CR-00066 — Functional Design

## ⛔ Docker is off-limits

(Standard policy. Testcontainer fixtures in tests are exempt.)

## ⛔ Migrations: agents generate, daemon applies

This item adds three new columns across two tables and seeds known model context limits. The migration file is generated by the database agent and applied by the daemon — never by the agent directly.

## Why

When an agent runs out of context window space mid-step, the run crashes with a cryptic error message and no prior warning. This happened during CR-00064 S01: the Pi agent consumed about 110,000 of MiniMax-M2.7's 200,000-token context window, then hit the limit and died silently — leaving a zero-byte log file and no visible clue that context pressure was building. Operators had no way to see this coming. This change gives operators a live, at-a-glance indicator of how much context each running step is consuming, so they can anticipate crashes before they happen.

## What Changed (for the User)

- The step table in any item detail page now has a "Context" column immediately to the right of the "Logs" column.
- For steps that used the Pi runtime, each row shows a small progress bar with a percentage indicating how much of the model's context window was consumed. The bar is color-coded: green for comfortable usage (under 61%), yellow for elevated usage (61–85%), and red for critical usage (above 85%).
- The figure updates automatically while a step is running, so operators can watch context pressure rise in near real time.
- For completed or failed steps, the bar shows the highest context usage reached during the entire run — even if the context was compacted and reset partway through.
- For models whose context limit is not yet configured, the column shows the raw token count without a percentage.
- For pending steps or steps that ran with Claude or OpenCode (where token data is not yet collected), the column shows a dash.

## How It Behaves

When a user opens an item's detail page, the Context column is always visible in the step table. A step running with Pi at 45% context shows a green bar labelled "45K / 200K (45%)". If context climbs past 61%, the bar turns yellow — a heads-up that the agent is in the high-usage zone. Above 85%, the bar turns red, signalling that a crash is likely if the step continues without context compaction. After a Pi compaction event, the current-usage value may drop, but the bar continues to reflect the all-time peak for that run so operators can see how close the edge was. A completed step that peaked at 150K / 200K stays yellow permanently, giving a historical record of context pressure for that execution.

## Out of Scope

- Context tracking for Claude and OpenCode runtimes (token data is not available in their log files in this release).
- Alerting or automatic step cancellation when context usage crosses a threshold.
- Per-project configuration of color thresholds.

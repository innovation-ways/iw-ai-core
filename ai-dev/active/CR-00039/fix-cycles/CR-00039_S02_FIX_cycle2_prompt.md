### 4. Status modifier completeness ⚠️ ONE GAP — HIGH

- **File**: `dashboard/templates/components/step_pipeline.html`, lines 12–16
- **Finding**: The Jinja2 `pill_status` variable maps `step.status == 'in_progress'` → `'in-progress'` (hyphen), but the CSS stylesheet defines `.iw-pipeline-pill--in_progress` (underscore, line 400). There is **no CSS class** for `iw-pipeline-pill--in-progress` (hyphen), so in-progress steps will never receive the pulse animation.
- **Impact**: `in_progress` steps will render with the `pending` fallback styles (no animation, wrong background color) until the CSS class name mismatch is fixed.
- **Affected statuses**:
  - `completed` ✅ → `.iw-pipeline-pill--completed`
  - `in_progress` ❌ → template emits `--in-progress`, CSS has `--in_progress`
  - `failed` ✅ → `.iw-pipeline-pill--failed`
  - `needs_fix` ✅ → also maps to `--failed`
  - `skipped` ✅ → `.iw-pipeline-pill--skipped`
  - `pending` ✅ → `.iw-pipeline-pill--pending`

#### Suggested Fix (pick one):

**Option A — Rename CSS class to hyphen** (cleaner, matches HTML naming conventions):
```css
/* line 400 — change underscore to hyphen */
.iw-pipeline-pill--in-progress { background: var(--primary); color: #fff; animation: pulse 1.4s ease-in-out infinite; }
```

**Option B — Rename template output to underscore** (preserves existing CSS):
```jinja
{% set pill_status = 'completed' if step.status == 'completed'
                 else 'in_progress' if step.status == 'in_progress'   {# underscore #}
                 else 'failed' if step.status in ('failed', 'needs_fix')
                 else 'skipped' if step.status == 'skipped'
                 else 'pending' %}
```

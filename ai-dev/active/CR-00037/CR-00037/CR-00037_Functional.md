# CR-00037 — Functional Design

## ⛔ Docker is off-limits

Standard policy. No Docker work in this CR.

## ⛔ Migrations: agents generate, daemon applies

Standard policy. This CR adds, modifies, or removes no migrations.

## Why

When the Files-tab feature was implemented, the implementing frontend agent had to start a third-party diff-rendering library, was given no concrete call shape in its instructions, and guessed wrong — burning three browser-verification fix cycles, about three quarters of an hour of automation time, before stumbling onto the right call. The post-mortem flagged the underlying problem as a missing reflex: the frontend agent had no standing rule to confirm a third-party library's actual exported surface before drafting code against it. Rather than patch a single already-shipped feature's instructions, we are adding that reflex once, in the frontend agent's definition itself, so every future frontend step across every managed project inherits it.

## What Changed (for the User)

Nothing changes in the running product. Operators, users, and dashboard viewers will see exactly the same dashboards, the same diffs, the same templates as before. The change is in the standing instructions that govern the frontend implementation agent: a short new step in its workflow that says, in effect, "before you call into a third-party UI library, confirm what it actually exposes — don't guess from its name." Once merged, that rule will be picked up by every managed project the next time someone runs the standard agent sync.

## How It Behaves

A reader who opens the frontend agent's definition will see, between the existing "study existing patterns" and "apply test-driven development" workflow steps, a new numbered step instructing the agent to grep the bundled file or read its type definitions before assuming a library function or factory exists. The step also carries one sentence of context naming the historical incident that motivated it, so future maintainers understand why it is there. Anyone synthesising a fresh frontend feature in iw-ai-core, IW-AI-DEV, InnoForge, or any other managed project will benefit from this reflex without having to remember to mention it in their per-step prompts.

## Out of Scope

- The product UI, server, database, and any test suite — none of those are touched.
- The historical run of the original feature (F-00079) — its work is already merged and is not being re-executed; its per-step prompts are not edited.
- Sync-generated copies under `.claude/agents/` and `.opencode/agents/` — those are regenerated by `iw sync-agents` and are not modified directly by this CR.
- Propagation of the change to IW-AI-DEV and InnoForge repos — handled as a follow-up after merge, not part of this CR.

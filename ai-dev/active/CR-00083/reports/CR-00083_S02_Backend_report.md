## Makefile Target: `make test-perf-daemon`

```makefile
test-perf-daemon:  ## Run daemon poll-loop performance budget test
    # --benchmark-compare uses pytest-benchmark's query() glob logic: 'daemon' expands
    # to <machine-id>/daemon*.json but saved files are 0002_daemon.json (with a prefix).
    # Reference the specific file directly to avoid the glob mismatch.
    uv run pytest tests/perf/test_daemon_poll_loop.py \
        -v --benchmark-compare=0002_daemon \
        --benchmark-compare-fail=mean:25% \
        --benchmark-storage=file://tests/perf/baselines \
        -m perf --no-cov
```

**pytest-benchmark `compare` vs `save` name resolution note:** The `--benchmark-save=daemon` flag creates `0002_daemon.json` (with a numeric prefix). When `--benchmark-compare=daemon` is passed, `pytest-benchmark`'s `query()` method searches for `<machine-id>/daemon*.json`, which does NOT match `0002_daemon.json`. The Makefile therefore uses `--benchmark-compare=0002_daemon` to reference the specific baseline file.

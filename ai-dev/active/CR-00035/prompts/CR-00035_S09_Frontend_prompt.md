## Requirements

### 1. Live Log card (visible while running)

```html
{% if job.job_type.value == 'doc_generation' and job.status == 'running' %}
<div class="bg-card border border-border rounded-lg p-4 mb-6">
  <div class="flex items-center justify-between mb-3">
    <h2 class="text-sm font-medium text-foreground">Live Log</h2>
    <span class="text-xs text-muted-foreground">streaming…</span>
  </div>
  <div hx-ext="sse"
       sse-connect="/project/{{ current_project.id }}/jobs/doc_generation/{{ raw.id }}/log/stream"
       sse-swap="message"
       hx-target="#doc-job-log"
       hx-swap="beforeend">
    <pre id="doc-job-log"
         class="font-mono text-xs whitespace-pre-wrap break-words bg-muted/50 rounded p-3 overflow-auto"
         style="max-height: 32rem;"
         aria-live="polite"
         aria-label="Live agent log"></pre>
  </div>
</div>
{% endif %}
```

Use `raw.id` (the UUID), not `raw.public_id` or `job.job_id`, in the URL — that matches the file-naming convention. The S07 endpoint accepts both, but the UUID is what the file is keyed by.

`aria-live="polite"` so screen readers announce new content non-disruptively. `aria-label` for the region.

### 6. Regression guard

The existing rendering for `job.job_type.value == 'doc_generation'` (Parameters card lines 95–132 and Error card lines 244–250) MUST continue to render correctly. Specifically:

- A failed job with `error="generation timeout after 15 minutes"` must still show the existing Error card AND now also the Execution Report card AND the Captured log fallback.
- A queued job (status `queued`, agent_pid null, no log file yet) must show the Parameters card and nothing else new (no Live Log card, no Execution Report card, no log link).

## Project Conventions

Read `dashboard/CLAUDE.md`:

- Tailwind classes only. No new CSS files. No inline `style=` except the `max-height` exception used here.
- Avoid dynamic class names (JIT purge breaks them).
- `make css` regenerates `dashboard/static/styles.css`.
- Templates extend `base.html`; fragments don't.
- htmx for AJAX; don't sprinkle ad-hoc JS.

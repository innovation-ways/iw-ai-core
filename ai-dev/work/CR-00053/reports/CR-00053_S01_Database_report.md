## Notes

- The partial unique index was correctly auto-generated by Alembic — no manual `postgresql_where` edit was needed (the `postgresql_where` argument was preserved in both `create_index` and `drop_index` calls).
- `idempotency_key` is nullable; the partial index `WHERE idempotency_key IS NOT NULL` ensures the unique constraint only applies when a key is provided, leaving the no-key path (AC1) unaffected.
- Composite PK on `(prefix, number)` makes this table a permanent audit log — the same number cannot be allocated twice for the same prefix.
- `project_id` column is nullable for forward compatibility (global-allocation paths may pass NULL).
- The `created_at` column uses `server_default=text("now()")` matching the existing `IdSequence.next_number` pattern, avoiding the need to set it in Python.
